# Artist Tagline / Bio Split Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Separar el subtítulo del hero (`tagline`) de la bio narrativa (`description`) en el modelo `Artist`, eliminando la duplicación visual entre el hero y la sección "La historia" que apareció después de importar bios largas desde los profiles del Researcher.

**Architecture:** Aditivo: nuevo campo `tagline String?` en `Artist` (Prisma). El hero renderiza `tagline` solo si está presente (sin fallback al `description`). `BioSection` deja de concatenar copy hardcodeado y usa solo `description`. Admin form gana un input. Importer nunca toca `tagline`. Backfill SQL recupera los 3 taglines viejos.

**Tech Stack:** Next.js 16 (App Router), Prisma 5 (Postgres / Supabase), React 19, server actions, zod 4.

**Working tree:** `/Users/christianfranciscogarcia/Proyects/Privada/artist-booking-app/.claude/worktrees/tagline-split` on branch `bg/artist-tagline-split`. `node_modules` and `.env` are symlinked from the main checkout.

**Spec:** `docs/superpowers/specs/2026-05-15-artist-tagline-split-design.md`

**Verification model:** Este proyecto no tiene framework de tests automatizados (no `test` script en `package.json`, no archivos de test). Cada tarea termina con typecheck/build cuando aplica, y la verificación funcional final está en la Tarea 10 (manual UI walk en `npm run dev`).
